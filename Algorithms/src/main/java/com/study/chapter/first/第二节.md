### 1.2.1.1 抽象数据类型的API
尽管数据类型定义的基础是一组值的集合，但是API可见的是对它的操作，而非它的内容。因此抽象数据类型的定义和静态方法库有些许相似
+ 两者实现均为java类
+ 实例方法可接受参数
+ 可能返回值，也能不返回

不同点：
+ API中可能会出现若干个名称和类名相同且没有返回值的函数，也就是构造
+ 实例方法不需要static关键字
+ 某些方法的存在是为了尊重Java的习惯

API：
    
    | public class Counter       |
    |       | Counter(String id) |       |
    | void  | increment()        |       |
    | int   | tally()            |       |
    | String| toString()         |       |

对象是能够承载数据类型的值的实体。对象有三大特性，状态，标识，行为。
